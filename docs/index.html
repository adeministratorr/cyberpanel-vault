Server-Level Protection for CyberPanel

Encrypted backup chains with a restore path you can actually operate.

CyberPanel Vault covers the server layer that website-only backups miss. It combines weekly full backups, incremental chains, encrypted Google Drive uploads, and a CyberPanel-ready job UI for controlled backup and restore runs.

/home, mail, database dumps, CyberPanel configuration, OpenLiteSpeed state, DNS, SSL, cron, firewall, and service-level host configuration.

Built for host recovery, not just website snapshots

CyberPanel Backup V2 is useful for site data. CyberPanel Vault focuses on the host-level layer and pairs that with a restore flow that understands weekly full and incremental chains.

Backup engine

  • Weekly full archives plus incremental follow-up runs
  • Full database dump on each execution
  • MyISAM-aware dump mode switching

Security

  • Encrypted archives before off-site upload
  • Checksums for integrity verification
  • Credential handling without leaking DB passwords into process lists

Recovery

  • Restore script applies the right chain in order
  • Server configs and data restored together
  • Job-oriented flow for controlled admin operations

Panel integration

  • Django app skeleton for CyberPanel integration
  • Backup and restore actions from a simple interface
  • Status and log driven operations

Workflow

The backup chain is intentionally conservative: full server coverage, encryption before upload, and deterministic restore sequencing.

1

Collect

Gather host data, application files, mail state, and a full database dump into a staged backup set.

2

Package

Build a full or incremental archive chain depending on the last successful full run and current schedule.

3

Encrypt

Encrypt the resulting archive and publish it to Google Drive through `rclone` with checksum metadata.

4

Restore

Download the selected chain, verify integrity, decrypt, and replay full plus incremental archives before service recovery.
