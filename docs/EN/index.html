Ready flow for GitHub publishing, GitHub Pages, and server deployment

This page collects the practical steps for publishing CyberPanel Vault to GitHub, enabling GitHub Pages from the repository, and preparing the backup and restore workflow on the target server. Publisher reference: Adem YÜCE - ademyuce.tr.

Recommended repository identity

  • Project name: CyberPanel Vault
  • Repository name: cyberpanel-vault
  • Website: https://ademyuce.tr
  • Pages URL: https://adeministratorr.github.io/cyberpanel-vault/

GitHub repository metadata

  • Description: CyberPanel server backup and restore manager with weekly full backups, incremental chains, Google Drive support, encryption, and a CyberPanel-ready UI.
  • Topics: cyberpanel, backup, restore, google-drive, rclone, incremental-backup, server-management, django, openlitespeed, devops

Push with Git

git add .
git commit -m "Publish GitHub Pages landing page"
git push origin main

Enable Pages

  1. Open the Settings page.
  2. Go to the Pages section.
  3. Select Deploy from a branch.
  4. Choose branch main and folder /docs.
  5. Save and verify the published URL.

SEO and GEO

  • docs/index.html includes title, description, canonical, Open Graph, and Twitter card metadata.
  • geo.region and geo.placename metadata are defined.
  • docs/robots.txt and docs/sitemap.xml are included in the published Pages site.

Server deployment notes

  • cyberpanel_full_backup.sh and cyberpanel_restore.sh must run as root on the target Linux server.
  • The server must have rclone, mysqldump, openssl, and the required service configuration paths.
  • Recommended encryption password file: /root/.config/cyberpanel-backup/encryption.pass
  • Always validate restore runs on staging before production use.

CyberPanel integration

The serverBackupManager/ directory is a Django app skeleton. It can be mounted into a CyberPanel-compatible plugin structure to trigger backup and restore jobs from an admin interface.
