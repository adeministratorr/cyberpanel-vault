Practical installation flow for host-level backup and restore

This page collects the practical steps for installing CyberPanel Vault on a CyberPanel host, running the first backup, validating restore chains, and wiring the project into a CyberPanel-style admin interface. Publisher reference: Adem YÜCE - ademyuce.tr.

Requirements

  • Linux server with root access
  • rclone, mysql, mysqldump, openssl, tar, gzip, sha256sum, flock, rsync
  • /etc/cyberpanel/mysqlPassword
  • A working Google Drive rclone remote
  • /root/.config/cyberpanel-backup/encryption.pass

Quick install

git clone https://github.com/adeministratorr/cyberpanel-vault.git /opt/cyberpanel-vault
cd /opt/cyberpanel-vault
install -m 750 cyberpanel_full_backup.sh /usr/local/bin/cyberpanel_full_backup.sh
install -m 750 cyberpanel_restore.sh /usr/local/bin/cyberpanel_restore.sh
mkdir -p /root/.config/cyberpanel-backup /var/lib/cyberpanel-backup
chmod 700 /root/.config/cyberpanel-backup /var/lib/cyberpanel-backup

First backup

printf '%s\n' 'REPLACE_WITH_A_LONG_RANDOM_SECRET' >/root/.config/cyberpanel-backup/encryption.pass
chmod 600 /root/.config/cyberpanel-backup/encryption.pass
BACKUP_MODE=full /usr/local/bin/cyberpanel_full_backup.sh

Daily usage

  • Automatic flow: BACKUP_MODE=auto /usr/local/bin/cyberpanel_full_backup.sh
  • Force a full backup: BACKUP_MODE=full /usr/local/bin/cyberpanel_full_backup.sh
  • Force an incremental backup: BACKUP_MODE=incremental /usr/local/bin/cyberpanel_full_backup.sh
  • Recommended cron entry: 0 3 * * * BACKUP_MODE=auto /usr/local/bin/cyberpanel_full_backup.sh

Restore

  • Validation and planning: /usr/local/bin/cyberpanel_restore.sh --target-file FILE_NAME
  • Real restore: /usr/local/bin/cyberpanel_restore.sh --target-file FILE_NAME --confirm-host "$(hostname -f)" --apply
  • Partial restore flags include --skip-db, --skip-files, --skip-configs, and --skip-services.
  • Always validate a full restore on staging before production use.

CyberPanel integration

The serverBackupManager/ directory is a Django app skeleton. It can be mounted into a CyberPanel-compatible plugin structure to trigger backup and restore jobs from an admin interface. For the full walkthrough, open the installation and usage page.
